前言#

博客基本搭建好了，不过还有不少细节需要完善一下。浏览了一些他人精致的博客网站，发现还有很多值得学习的地方。接下来就从 评论系统 、侧边栏目录 以及 中文字数统计 三个方面入手，进行一些美化。

评论系统#

搭建参考教程：一步一步教你在Jekyll博客添加评论系统 - 一之笔的博客

之前在其实使用过 Disqus，但是几经比较还是觉得 Gitalk 更加美观些，而且使用 GitHub 账号登录也比较方便，支持 markdown 语法这一点也深得我意 （其实根本没有人会评论）。所以最后还是关掉了 Disqus，着手开始搭建 Gitalk 啦 ~

申请 GitHub OAuth Application#

在 GitHub 中点击头像下拉菜单，选择 Settings，然后点击左侧的 Developer settings ，接下来点击 OAuth Apps > Register a new application，开始填写你的信息：

点击 Register application 后，就会得到你的 Client ID 和 Client Secret。

在 Jekyll 中添加 Gitalk#

在你需要添加评论系统的地方，一般是 _layout/ 目录下的 post.html：

在 Post Header 中，引入 gitalk.css 和 gitalk.min.js ：

 <link rel="stylesheet" href="https://unpkg.com/gitalk/dist/gitalk.css">
 <script src="https://unpkg.com/gitalk/dist/gitalk.min.js"></script>

在 Post Container 尾部，添加容器和 JavaScript 代码：

<!-- Gitalk 评论框 start -->
<div class="comment">
    <div id="gitalk-container"></div>
    <script>
        var gitalk = new Gitalk({
            clientID: 'GitHub Application Client ID',
            clientSecret: 'GitHub Application Client Secret',
            repo: 'GitHub repo',
            owner: 'GitHub repo owner',
            admin: ['GitHub repo owner and collaborators, only these guys can initialize github issues'],
            id: location.pathname,      // Ensure uniqueness and length less than 50
            distractionFreeMode: false  // Facebook-like distraction free mode
        })
        gitalk.render('gitalk-container')
    </script>
</div>
<!-- Gitalk 评论框 end -->

具体设置参考 Gitalk 的中文说明。

保存后，提交代码就行啦。

侧边栏目录#

这个问题有点乌龙，其实我 Fork 的 Hux Blog 主题中已经自带 catalog，我在 post.html 中也看到了相关的 div 块：

<!-- Side Catalog Container -->
{% if page.catalog %}
<div class="
            col-lg-2 col-lg-offset-0
            visible-lg-block
            sidebar-container
            catalog-container">
    <div class="side-catalog">
        <hr class="hidden-sm hidden-xs">
        <h5>
            <a class="catalog-toggle" href="#">CATALOG</a>
        </h5>
        <ul class="catalog-body"></ul>
    </div>
</div>
{% endif %}

因此，只要在 .md 文件中添加一行代码即可：

---
layout:     post
title:      "……"
subtitle:   "……"
date:       ……
author:     "Thistledown"
header-img: "……"
# 添加下面这行代码
catalog: true  
tags: ……
---

添加中文字数统计#

想要统计文章的字数，首先我们需要得到文章的内容，即 page.content。

Page-Variables - Jekyll：
page.content : The content of the Page, rendered or un-rendered depending upon what Liquid is being processed and what page is.

而模板语言 Liquid 是 Jekyll 的特色之一，它有三个主要部分：对象，标签和过滤器。了解更多：jekyllrb.com。

过滤器 filter 更改 Liquid 对象的输出，并由 | 分隔。其中恰好就有现成的字数统计过滤器 number_of_words，然而，它用来统计英文非常方便，统计中文却错误频出，往往一句或者一段话只统计为一个词。推测这种方法可能是用空格来分割字符进行统计，这明显不符合我们中文写作的实际，因此需要稍微走点弯路，用其他的方法来解决。

我们需要的 Liquid Filter 有：

  • strip_html：从字符串中删除所有的 HTML 标记
  • strip_newlines：从字符串中删除所有换行符
  • split：是用参数作为分隔符将输入字符串分割为数组
  • size：返回字符串中的字符数或数组中的项数

去掉文章内容中的 HTML 标签和换行符，将所有字符都单个地存储在数组中，只要统计数组的 size，就能够统计出文章字数。因此，代码也就非常简单：

<!-- Number of words -->
{% raw %}{{ page.content | strip_html | strip_newlines | split: "" | size }}{% endraw %}
